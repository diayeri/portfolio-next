## 🚀 Key Achievements

공연 예술 통합 전산망(Kopis) API를 활용한 공연 정보 공유 및 리뷰 플랫폼 **Muna**의 UX 기획과 전체 UI 디자인(90%), 그리고 프론트엔드 개발을 담당하였습니다. 프로젝트 초기 단계에서 실무 경험을 바탕으로 세분화된 **WBS**(Work Breakdown Structure)를 직접 설계하고 일정을 관리하여, 기획부터 개발 환경 세팅까지 단 1주일 만에 완료하는 효율적인 프로세스를 주도했습니다.

**Atomic Design** 철학을 기반으로 디자인 시스템을 컴포넌트 단위로 체계적으로 정리하였습니다. 이를 통해 개발 과정에서 발생할 수 있는 UI 수정을 최소화하고 재작업으로 인한 시간 낭비를 방지하여, 팀원들이 핵심 기능 구현에만 몰입할 수 있는 표준화된 협업 환경을 구축했습니다.

- **디자인 시스템 설계 전담, UI 디자인 90% 담당:** Figma와 React 컴포넌트를 동기화한 시스템 구축으로 개발 효율 극대화
- **실무 기반의 린(Lean) 프로젝트 주도:** WBS 설계를 통한 병렬 작업 구조화로 안정적인 개발 기간 확보
- **최우수상 수상 및 기술력 인정:** 프로젝트 완료 후 완성도를 인정받아 최우수상을 수상하였으며, 실무 수준의 결과물이라는 높은 평가를 받음
- **데이터 구조 최적화:** Kopis API의 제약사항을 분석하고 Firebase와 Redux를 활용해 사용자 활동 기반 정렬 로직 구현

## 🛠 Challenges & Technical Excellence

### 효율적인 협업을 위한 시스템 가이드 정의

기획 변경이나 UI 수정에 따른 반복적인 재개발을 방지하기 위해 프로젝트 시작 전 **디자인 시스템**을 선제적으로 정의했습니다. **Atomic Design** 패턴에 따라 가장 작은 단위인 버튼, 입력창부터 복잡한 카드 형태까지 체계적으로 설계하였으며, 이를 코드 단의 **공통 컴포넌트**와 동기화했습니다.

이러한 설계 덕분에 개발 중 UI 수정 비용을 획기적으로 낮출 수 있었고, UI 디자인 결과물이 코드에 즉각적으로 반영되어 높은 디자인 완성도를 유지할 수 있었습니다. 이는 팀원들이 복잡한 레이아웃 고민 없이 기능 구현에만 집중할 수 있는 튼튼한 기반이 되었습니다.

### API 제약사항을 극복한 데이터 및 상태 관리

Kopis Open API가 최신순 정렬 등 필수적인 기능을 제공하지 않는 한계를 파악하고, 이를 **Firestore DB**와 **Redux Toolkit**으로 보완하는 아키텍처를 설계했습니다. `createAsyncThunk`를 활용해 Firebase와의 비동기 통신을 전역 상태로 관리하며, API가 제공하지 못하는 정렬 로직을 프론트엔드 단에서 안정적으로 구현했습니다.

또한, **Firebase Emulator**를 도입하여 로컬 환경에서도 클라우드 서비스와 동일한 환경을 구축함으로써 데이터 연동 오류를 사전에 차단하고 개발 생산성을 높였습니다.

## 📊 Results & Lessons Learned

실무 경험을 바탕으로 **디자인 시스템 구축과 프론트엔드 개발, 그리고 전체 업무 일정 관리**를 주도하며 프로젝트의 효율성을 극대화한 경험이었습니다. 특히 1개월이라는 짧은 기간 내에 핵심인 '프론트엔드 개발'에 집중할 수 있도록, 기획과 디자인 단계에서 컴포넌트 단위의 치밀한 설계를 선행하여 이후의 수정을 최소화했습니다. 그 결과, 작업 방향이 뒤바뀌거나 불필요한 재작업을 최소화하고 전체 일정 중 2주 이상의 충분한 개발 기간을 확보할 수 있었습니다.

프로젝트 초기 일주일간 페이지별 기획과 디자인을 리드함과 동시에, 다른 팀원들이 백엔드 API 세팅 등 병렬적으로 작업할 수 있도록 업무 순서를 체계적으로 조율 및 주도했습니다. 이러한 효율적인 협업 환경 구축 덕분에 프로젝트의 완성도를 높일 수 있었고, 최종 심사에서 **프로젝트 최우수상**을 수상하며 실무 수준의 프로세스를 검증받았습니다. 단순히 기능을 구현하는 것을 넘어, 팀 전체의 흐름을 조율하고 병목을 사전에 해결하는 과정에서 협업을 리드하는 UI 개발자로서의 역량을 깊이 다질 수 있었습니다.
