## 🚀 Key Achievements

본 프로젝트에서 **전체 페이지의 UI 개발 (100%)**을 전담하고, **모바일 UI 디자인**에 참여했습니다. 단순한 화면 구현을 넘어 설계부터 최종 테스트까지 개발의 전 과정을 주도하였으며, **Atomic Design** 철학을 참고하여 **React와 TypeScript** 기반의 컴포넌트 아키텍처를 직접 구축하고 관리했습니다.

업무 병목 구간에서 선제적으로 디자인과 퍼블리싱을 동시에 작업하는 방식을 제안하여 전체 개발 일정을 단축시켰으며, **Storybook**을 통한 문서화로 디자이너 및 프론트엔드 개발자와의 협업 효율을 극대화했습니다.

## 🛠 Challenges & Technical Excellence

### 복합적인 거래 환경을 고려한 반응형 UI 설계

선물 거래 특성상 다양한 데이터를 한 화면에 효율적으로 배치하는 것이 중요했습니다. 이를 위해 처음부터 **Atomic Design** 패턴을 적용하여 재사용 가능한 최소 단위 컴포넌트를 먼저 제작하고, 이를 조합하여 **Props 기반의 동적 React 컴포넌트** 구조를 완성했습니다.

특히 **차트 확장 및 축소 기능** 구현에 공을 들였습니다. 화면이 작은 모바일에서는 핵심 거래창에 집중하도록 설계하되, 태블릿과 PC 뷰에서는 반응형 사이즈에 따라 **Short / Long 거래창을 동시에 확인**할 수 있도록 구성했습니다. 이를 통해 사용자가 기기 환경에 구애받지 않고 빠르고 편리하게 코인 선물 거래를 진행할 수 있는 최적의 UI를 제공했습니다.

### 능동적인 협업과 공정 리스크 관리

프로젝트 진행 중 디자인 업무 부하로 인해 퍼블리싱과 개발 단계에서 대기 시간이 발생하는 병목 현상이 있었습니다. 협업자 간 업무 부담을 줄이고 팀의 생산성을 높이기 위해 **모바일 및 태블릿 뷰의 UI 설계를 선제적으로 진행**하고 이후에 디자인을 보완하는 방식을 제안했습니다. 디자이너의 동의 하에 직접 디자인과 퍼블리싱을 병행한 뒤, 사후에 디자이너와 함께 디테일을 수정하는 방식을 도입했습니다. 이 과정은 전체적인 개발 속도를 획기적으로 개선하는 계기가 되었습니다.

또한, 모든 공통 컴포넌트를 **Storybook**으로 문서화하여 팀에 공유하여 협업 효율성을 높였습니다.

- **디자이너의 입장:** 개발자의 도움 없이도 작업된 UI를 실시간으로 확인하고 피드백을 전달 가능
- **프론트엔드 개발자의 입장:** 정리해둔 컴포넌트들을 상태별로 즉시 확인하고 기능을 추가할 수 있어, 컴포넌트 재사용 및 기능 구현 속도가 크게 향상

## 📊 Results & Lessons Learned

Klaybay에서 시도했던 **선제적 마크업과 React 기반 UI 구현 방식**을 더욱 안정적으로 내재화하여 팀워크를 극대화한 프로젝트입니다. **Atomic Design 패턴**을 기반으로 재사용 가능한 컴포넌트 구조를 설계하고, **Tailwind CSS**와 **TypeScript**를 엄격하게 적용하여 리소스 효율화와 런타임 안정성을 동시에 확보했습니다. 여기에 **Storybook을 통한 UI 문서화**와 **파일럿 프로젝트 검증**을 더해 팀 전체의 개발 퍼포먼스를 높이는 시스템을 구축했습니다.

이러한 시도들은 함께 협업하는 동료들의 고충을 덜어주기 위한 고민에서 시작되었습니다. 협업자의 병목을 해결하기 위해 시작한 노력이 결국 저의 기술적 영역을 확장하고 새로운 성장을 이끄는 강력한 동기부여가 되었으며, 이는 곧 작업의 즐거움으로 이어졌습니다. **'내가 조직에 긍정적인 영향을 주고 있다'는 실감**은 제가 더 몰입하게 만드는 가장 큰 원동력이 되었고, 설계를 넘어 개발 전반을 아우르는 경험을 통해 팀과 개인이 동반 성장하는 협업의 가치를 깊이 체감할 수 있었습니다.
