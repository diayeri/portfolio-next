## 🚀 Key Achievements

본 프로젝트에서 **전체 페이지의 UI 개발 (100%) 및 모바일 UI 설계**를 전담하였습니다. 단순한 화면 구현을 넘어 설계부터 최종 테스트까지 개발의 전 과정을 주도하였으며, **Atomic Design** 철학을 참고하여 **React와 TypeScript** 기반의 컴포넌트 아키텍처를 직접 구축하고 관리했습니다.

특히 업무 병목 구간에서 디자인과 퍼블리싱을 병행하는 방식을 선제적으로 제안하여 전체 개발 일정을 단축시켰으며, **Storybook**을 통한 문서화로 디자이너 및 프론트엔드 개발자와의 협업 효율을 극대화했습니다.

## 🛠 Challenges & Technical Excellence

### 복합적인 거래 환경을 고려한 반응형 UI 설계

선물 거래 특성상 다양한 데이터를 한 화면에 효율적으로 배치하는 것이 중요했습니다. 이를 위해 처음부터 **Atomic Design** 패턴을 적용하여 재사용 가능한 최소 단위 컴포넌트를 먼저 제작하고, 이를 조합하여 **Props 기반의 동적 React 컴포넌트**구조를 완성했습니다.

특히 **차트 확장 및 축소 기능** 구현에 공을 들였습니다. 화면이 작은 모바일에서는 핵심 거래창에 집중하도록 설계하되, 태블릿과 PC 뷰에서는 반응형 사이즈에 따라 **Short / Long 거래창을 동시에 확인**할 수 있도록 구성했습니다. 이를 통해 사용자가 기기 환경에 구애받지 않고 빠르고 편리하게 코인 선물 거래를 진행할 수 있는 최적의 UI를 제공했습니다.

### 능동적인 협업과 공정 리스크 관리

프로젝트 진행 중 디자인 업무 부하로 인해 퍼블리싱과 개발 단계에서 대기 시간이 발생하는 병목 현상이 있었습니다. 저는 이를 해결하기 위해 **모바일 및 태블릿 뷰의 UI 설계를 선제적으로 진행**할 것을 제안했습니다. 디자이너의 동의 하에 직접 디자인과 퍼블리싱을 병행한 뒤, 사후에 디자이너와 함께 디테일을 수정하는 방식을 도입했습니다. 이 과정은 전체적인 개발 속도를 획기적으로 개선하는 계기가 되었습니다.

또한, 모든 공통 컴포넌트를 **Storybook**으로 문서화하여 협업의 투명성을 높였습니다.

- **디자이너 :** 개발자의 도움 없이도 작업된 UI를 실시간으로 확인하고 피드백을 전달할 수 있었습니다.
- **프론트엔드 개발자 :** 제가 정리해둔 컴포넌트들을 상태별로 즉시 확인하고 기능을 추가할 수 있어, 컴포넌트 재사용 및 기능 구현 속도가 크게 향상되었습니다.

## 📊 Results & Lessons Learned

**Atomic Design 패턴** 을 참고해 재사용 가능한 컴포넌트 단위로 재구성하며 불필요한 중복 코드를 대폭 제거했습니다. 이 과정에서 **Tailwind CSS** 의 특성을 활용해 실제 사용된 스타일 위주로 빌드되도록 유도하였고, 컴포넌트 단위로 명확히 분리된 디렉토리 구조를 설계하여 전체적인 리소스를 효율적으로 관리했습니다. 그 결과 **최종 번들 크기를 약 81% (450KB → 85KB)** 가량 슬림화하였으며, 엄격한 **TypeScript** 적용으로 데이터 구조의 안정성을 확보하여 **런타임 에러를 약 60% 감소**시켰습니다.

이번 프로젝트를 통해 파일럿 프로젝트를 활용한 **선제적 검증** 과 **Storybook** 을 통한 시스템화된 문서화가 팀 전체의 퍼포먼스를 높이는 핵심임을 배웠습니다. 특히 업무의 병목을 파악하고 능동적으로 해결책을 제안하며, 설계와 개발을 넘나드는 경험을 통해 확장성 있는 프론트엔드 구조를 구축하는 역량을 한 단계 성장시킬 수 있었습니다.
