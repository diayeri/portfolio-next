## 🚀 Key Achievements

기존 모바일 후원 결제 과정에서 발생하는 사용자 이탈 문제를 해결하기 위해 **결제 UX 개선 프로젝트를 직접 제안하고 총괄**하였습니다. 타 기관 서비스 벤치마킹과 시나리오 분석을 통해 복잡한 입력 과정을 2단계로 재구성하였으며, 후원자 유형별 맞춤형 플로우를 설계했습니다.

개선 결과, 사용자 편의성이 크게 향상되어 **모바일 후원이 전년 대비 2배 증대**되는 실질적인 성과를 거두었습니다. 외주 개발사와 직접 협업하며 QA 및 최종 구현 결과까지 관리하여 설계 의도가 정확히 반영되도록 이끌었습니다.

### 주요 성과

- **결제 프로세스 최적화:** 개인/기업/단체 등 후원자 유형별 필수 정보만 노출하여 입력 장벽 완화
- **모바일 사용성 개선:** 터치 영역 확대, 폰트 조절, 스크롤 최소화를 통해 모바일 환경에 최적화된 UI 제공
- **데이터 기반 성과 달성:** UX 개선 후 모바일 채널을 통한 후원 성과 전년 대비 200% 달성
- **외부 개발사 협업 관리:** 전화 및 문서 기반의 상세 커뮤니케이션으로 결제 UI 테스트 및 수정 요청 총괄

## 🛠 Challenges & Technical Excellence

### 심리적 부담을 낮추는 단계별 입력 설계

한 페이지에 과도하게 집중되어 있던 입력 항목들을 분석하여 **2단계 구조**로 재배치했습니다. 1단계에서는 후원에 필요한 최소한의 기본 정보만 남기고, 결제 수단 등의 상세 정보는 2단계로 분리하여 사용자가 느끼는 심리적 부담을 줄였습니다.

'납부'와 '납입' 등 혼용되던 용어들을 정리해 직관성을 높였고, 선택 항목은 접기/펼치기 기능을 적용해 시각적 복잡도를 낮추었습니다. 이전 단계의 입력값이 자동으로 연결되도록 설계하여 중복 입력을 방지하고 결제 완료까지의 시간을 단축했습니다.

### 모바일 환경에 최적화된 인터페이스 구현

작은 화면에서의 터치 실수를 방지하기 위해 라디오 버튼을 큰 버튼 형태로 변경하고, 입력 필드 간격과 폰트 크기를 가독성 있게 조정했습니다. 불필요한 안내 문구 대신 Placeholder를 적극 활용하여 라벨 영역을 줄였고, 전체적인 페이지 길이를 단축해 스크롤의 피로도를 낮췄습니다.

후원관리 부서와 협의하여 필수 정보를 재정리하고, 결제 수단의 기본값을 '카드'로 통일하는 등 실무적인 효율성을 고려한 UI를 구축했습니다.

### 외부 개발사와의 긴밀한 구현 결과 관리

설계한 UX가 실제 서비스에 안정적으로 적용될 수 있도록 외부 개발사와 직접 소통하며 QA 과정을 관리했습니다. 문서 기반의 상세한 수정 요청서를 통해 결제 단계에서 발생할 수 있는 변수들을 사전에 점검했으며, 구현 과정에서의 기술적 제약 사항을 디자인적으로 보완하며 최종 결과물의 품질을 높였습니다.

## 📊 Results & Lessons Learned

QA 과정에서 발견한 작은 불편함을 놓치지 않고 개선 프로젝트로 연결하여 **모바일 후원 2배 증대**라는 비즈니스 성과를 이끌어낸 값진 경험이었습니다. 데이터를 바탕으로 문제를 정의하고, 유관 부서 및 개발사와의 협업을 통해 해결책을 실행에 옮기는 전체 과정을 주도하며 기획과 디자인을 아우르는 역량을 확인했습니다.

단순히 보기 좋은 디자인을 넘어, 서비스의 목적에 부합하는 **논리적인 설계**가 사용자 행동 변화와 성과에 얼마나 직접적인 영향을 미치는지 깊이 체감할 수 있었습니다.
